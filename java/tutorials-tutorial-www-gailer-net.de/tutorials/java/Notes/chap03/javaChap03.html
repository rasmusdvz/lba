Sowohl im Hauptspeicher als auch im Sekundärspeicher werden Information als Bitmuster gespeichert. Erinnern Sie sich aus dem Kapitel 2 was ein Bit ist:

Bit
Ein Bit ist ein einzelner An/Aus-Wert. Nur diese Werte sind möglich.

Diese zwei Werte haben auch andere Namen wie true/false (wahr/falsch) oder 1/0. Es gibt viele Möglichkeiten wie ein Bit implementiert werden kann. Zum Beispiel könnte ein Bit implementiert sein als:

  • Ein mechanischer, elektrischer Schalter (wie ein Lichtschalter).
  • Spannung einer Leitung.
  • Ein einzelner Transistor (verwendet im Hauptspeicher).
  • Ein winziger Teil auf der Oberfläche einer magnetischen Festplatte.
  • Ein winziger Teil auf der Oberfläche eines magnetischen Bands.
  • Ein gestanztes Loch in einer Karte.
  • Ein winziger Teil auf der lichtreflektierenden Oberfläche einer CD.
  • Teil eines Funksignals.
  • Viele, viele andere Möglichkeiten

Daher ist die jeweilige Implementation von Bits im Hauptspeicher und Sekundärspeicher unterschiedlich, aber logisch betrachtet speichern beide Arten von Speicher Bits.

(Denkfrage:) Ändert sich die Information, wenn ein Bit von einem Speichermedium auf ein anderes kopiert wird? Zum Beispiel ändert sich die Information, wenn ein durch Spannungshöhe implementiertes Bit einer Leitung auf einen winzigen Teil einer Festplatte kopiert wird?

Kopierte Information

Nein. Die verschiedenen Möglichkeiten ein Bit zu speichern ändern nicht die Information, die gespeichert wird.

Information, die in binärer Form gespeichert ist, ändert sich nicht, wenn sie von einem Medium (Speichermethode) auf ein anderes kopiert wird, und es kann eine unbegrenzte Anzahl solcher Kopien gemacht werden (denken Sie an die Vorteile des Binären). Dies ist ein äußerst mächtige Kombination. Sie mögen sich so daran gewöhnt haben, dass es Ihnen alltäglich erscheint. Aber wenn Sie ein Bild aus dem Internet herunterladen, werden die Daten Dutzende Mal kopiert, unter Verwendung einer Vielzahl von Speicher- und Übertragungsmethoden.

Es ist zum Beispiel wahrscheinlich, dass die Daten sich von einer magnetischen Festplatte aufmachen und dann in den Hauptspeicher eines Computers einer Website kopiert werden (was ein Spannungssignal zur Folge hat). Vom Hauptspeicher werden sie (wieder durch ein Spannungssignal) auf eine Netzkarte kopiert, die sie temporär in vielen Transistoren hält. Von dort werden sie als ein elektrisches Signal durch eine Leitung geschickt. Den Weg entlang zu Ihrem Computer, es kann Dutzende von Computern geben, die die Daten aus einem elektrischen Signal in den Hauptspeicher in Transistorform transformieren, dann zurück als elektrisches Signal auf ein anderes Kabel. Ihre Daten könnten sogar in ein Funksignal transformiert werden, zu einem Satelliten (mit seinen eigenen Computern) geschickt werden, und mit einem anderen Funksignal zur Erde zurückgeschickt werden. Letztendlich landen die Daten in Ihrer Videokarte (Transistoren), die sie in ein TV Signal für Ihren Bildschirm transformiert.

Der Punkt von allem dem ist, dass sich die tatsächliche Information (in diesem Beispiel das Bild) von einem Medium zum nächsten nicht ändert.

Analoge Daten (sich fortlaufend ändernde Signale) wie Schallplatten- oder Kassettenaufnahmen können ebenfalls von Medium zu Medium kopiert werden. Ändert sich die Information analoger Daten, wenn sie auf ein anderes Medium kopiert werden?

Eine Bit an Information ist so wenig, dass Computerspeicher üblicherweise in Gruppen von acht Bits organisiert ist. Jede 8-Bit Gruppe wird Byte genannt. Wenn Daten mehr als 8 Bits brauchen, wird eine ganze Anzahl von Bytes verwendet. Eine Byte ist ungefähr genug Speicher um ein einzelnes Zeichen zu enthalten.

Für Daten werden oft sehr viel mehr als acht Bits benötigt und es werden Tausende, Millionen oder sogar Milliarden von Bytes gebraucht. Diese Mengen haben Namen, wie in der Tabelle angegeben.

(Von der mittleren Spalte sollten Sie sich merken, dass ein Kilobyte 1024 Bytes sind.) Ein Kilobyte wird oft ein "K" und a Megabyte wird ein "Meg" genannt.

Es wird für Ihre berufliche Laufbahn sehr wertvoll sein, wenn Sie sicher wissen wie Zweier-Potenzen zu multiplizieren sind.

Hauptspeicher, Organisation

Hauptspeicher besteht aus einer langen Liste von Bytes. In den meisten modernen Computern besitzt jedes Byte eine Adresse, die verwendet wird um es zu lokalisieren. Die Abbildung zeigt einen kleinen Teil Hauptspeicher:

Jede Zelle der Abbildung stellt ein einzelnes Byte dar. Jedes Byte hat eine Adresse. In dieser Abbildung sind die Adressen die ganzen Zahlen (Integer) links von den Zellen: 0, 1, 2, 3, 4, ... und so weiter. Die Adressen der meisten Computerspeicher beginnen mit 0 und werden der Reihe nach erhöht bis jedes Byte eine Adresse hat.

Jedes Byte enthält ein Muster von acht Bits. Wenn der Computer eingeschaltet ist, enthält jedes Byte das eine oder das andere Muster, auch jene Bytes, die nicht gebraucht werden. (Denken Sie an das Wesen des Binären: wenn ein binäres Gerät arbeitet ist es entweder "an" oder "aus", niemals dazwischen.)

Die Adresse eines Bytes ist nicht Teil seines Inhalts. Wenn der Prozessor auf ein Byte an einer bestimmten Adresse zugreifen muss, "weiß" die Elektronik des Computers wie dieses Byte im Speicher zu finden ist.

Hauptspeicher (wie jeder Computerspeicher) speichert Bits als Muster. Das heißt, jede Speicherstelle besteht aus acht Bits und jedes Bit ist entweder "0" oder "1". Zum Beispiel zeigt die Abbildung die ersten paar Speicherbytes.

Speicher, Inhalte

Das Einzige, was an einer Speicherstelle gespeichert werden kann, sind acht Bits, jedes mit einem Wert von "0" oder "1". Die Bits an einer Speicherstelle werden die Inhalte dieser Stelle genannt.

Manchmal werden Leute sagen, dass jede Speicherstelle eine binäre 8-Bit Zahl enthält. Das ist OK, solange Sie daran denken, dass die "Zahl" verwendet werden könnte um ein Zeichen darzustellen oder irgendetwas anderes.

Denken Sie daran, dass es vom Zusammenhang abhängt, was ein bestimmtes Muster darstellt (das heißt, wie ein Programm es verwendet). Sie können kein beliebiges Bitmuster betrachten (wie jenes in der Abbildung) und sagen was es darstellt.

Daten, lesen Daten, schreiben

Der Prozessor kann zwei fundamentale Dinge im Hauptspeicher tun:

  1. Er kann ein Byte an einer gegebenen Speicherstelle schreiben.
    • Das bisherige Bitmuster an dieser Stelle wird zerstört.
    • Das neue Bitmuster wird für zukünftigen Gebrauch gespeichert.

  2. Er kann ein Byte an einer gegebenen Speicherstelle lesen
    • Der Prozessor bekommt das an dieser Stelle gespeicherte Bitmuster.
    • Die Inhalte an dieser Stelle werden NICHT geändert.

Wenn der Prozessor zum Beispiel ein Byte braucht, das an der Speicherstelle 5 gespeichert ist, kann er es lesen. Er bekommt das Byte "0110 1110" als die Daten, die er braucht (aber die Stelle 5 im Speicher ändert sich nicht).

Die meisten Prozessoren können mehr als ein einzelnes Byte zur gleichen Zeit schreiben und lesen. Das beschleunigt die Dinge. Aber die oben stehenden Operationen sind elementar. Sie haben vielleicht etwas über die neuen 64-Bit Prozessoren und 128-Bit Prozessoren gehört. Ein 64-Bit Prozessor kann gleichzeitig 64/8 = 8 Bytes lesen und schreiben.


Angenommen, dass der Prozessor später das Byte "1111 1111" speichern muss und dafür die Speicherstelle 7 nimmt. Wie wird sich der Hauptspeicher ändern?

Programme und Speicher

Die Antwort finden Sie in der Abbildung unten.

Der Prozessor hat ein Byte von Daten an die Stelle 7 geschrieben. Die ehemaligen Inhalte an dieser Stelle sind verloren. Der Hauptspeicher sieht nun wie in der Abbildung aus.

Wenn ein Programm läuft, verfügt es über einen Speicherabschnitt für die Daten, die es verwendet. Die Speicherstellen in diesem Abschnitt können so oft geändert werden, wie es das Programm benötigt. Wenn zum Beispiel ein Programm eine Liste von Zahlen addiert, wird die Summe im Hauptspeicher (wahrscheinlich unter Verwendung mehrerer Bytes) gehalten. Während der Summe neue Zahlen hinzugefügt werden, wird sie sich ändern und der Hauptspeicher wird sich ebenfalls ändern.

Andere Bereiche des Hauptspeicher könnten sich überhaupt nicht ändern, während ein Programm ausgeführt wird. Zum Beispiel ändern sich während der Laufzeit eines Programms die Befehle, die ein Programm ausmachen, (üblicherweise) nicht. Die Befehle eines laufenden Programms befinden sich im Hauptspeicher, also werden diese Stellen nicht geändert.

Wenn Sie ein Programm in Java (oder in einer der meisten anderen Sprachen) schreiben, müssen Sie die Speicherstellen und deren Inhalte nicht im Auge behalten. Es ist ein Grund für die Verwendung einer Programmiersprache diese Dinge automatisch zu tun.


(Rückblickende Frage:) Wo werden Programme und Daten für langfristige Speicherung aufbewahrt?

Festplatten

Im Sekundärspeicher (üblicherweise auf der Festplatte des Computersystems).

Festplatte

Die Festplatte eines Computersystems zeichnet Bytes auf einer magnetischen Oberfläche auf, vergleichbar mit der Oberfläche eines Tonbandes. Das Aufzeichnen (Schreiben) und Lesen der Daten geschieht mit einem Lese-/Schreibkopf, ähnlich dem, der bei einem Tonbandgerät verwendet wird.

Die Abbildung zeigt eine Platte und einen Lese-/Schreibkopf am Ende des beweglichen Arms. Der Arm bewegt sich entlang des Radius der Platte auf und ab. Da die Platte rotiert, wird er Daten in einer kreisförmigen Spur auf der Platte aufzeichnen. Später muss er an die richtige Stelle bewegt werden um die Daten zu lesen, dann muss er warten bis die rotierende Platte die Daten in Position gebracht hat. Genau wie bei einem Tonband können Daten gelesen werden ohne sie zu ändern. Wenn er neue Daten aufzeichnet, ersetzt er jedwede Daten, die früher an dieser Stelle aufgezeichnet worden waren. Im Gegensatz zum Tonband, berührt der Lese-/Schreibkopf nicht wirklich die Platte, sondern fliegt in geringer Höhe über sie hinweg.

Gewöhnlich enthält die Komponente eines Computersystems, die "Festplatte" genannt wird, viele einzelne Platten und Lese-/Schreibköpfe. Die Platten sind auf beiden Seiten mit einem magnetischen Material überzogen (daher bekommt jede Platte zwei Lese-/Schreibköpfe) und die Platten sind miteinander über eine Achse verbunden. Alle Platten und Köpfe sind in einem absolut staubfreien Metallgehäuse untergebracht. Da der Betrieb einer Festplatte mechanische Bewegung mit sich bringt (die sehr viel langsamer ist als elektronische Prozesse), ist das Lesen und Schreiben von Daten sehr viel langsamer als im Hauptspeicher.

(Denkfrage:) Warum berühren die Lese-/Schreibköpfe einer Festplatte nicht die Oberfläche?

Dateien

Da die Platte bei eingeschalteten Computer ständig rotiert wäre sie bald abgenutzt, wenn der Lese-/Schreibkopf Kontakt mit ihr hätte.

Datei, Definition

Festplatten (und andere sekundäre Speichermedien) werden für die langfristige Speicherung von großen Informationsblöcken verwendet, so wie Programme und Daten-Sets. Plattenspeicher wird gewöhnlich in Dateien organisiert.

Eine Datei ist eine Sammlung von Informationen, für die eine Name vergeben wurde und, die im Sekundärspeicher gespeichert wird. Die Information kann ein Programm oder Daten sein.

Die Art der Information in einer Datei ist die gleiche wie die jeder digitalen Information---sie besteht aus Bits, üblicherweise in 8-Bit Bytes gruppiert. Dateien sind häufig ganz schön groß; ihre Größe wird in Kilobyte oder Megabyte gemessen.

Wenn Sie noch nie mit Dateien in einem Computer gearbeitet haben, sollten Sie die Dokumentation Ihres Betriebssystems studieren oder einen Blick in ein Buch wie Windows NT für Dummies werfen (oder was für Ihren Computer angebracht ist).

Betriebssystem, Datei I/O

Es ist eine der Aufgaben eines Computerbetriebssystems Dateinamen und wo sie sich auf der Festplatte befinden im Auge zu behalten. Der Anwender kann zum Beispiel in DOS den Start des Programms HANSE folgendermaßen anfordern: 

C:\> HANSE.BAT

Das "C:\>" ist eine Eingabeaufforderung (Prompt); der Anwender hat "HANSE.BAT" eingegeben. Das Betriebssystem muss jetzt die Datei mit dem Namen HANSE.BAT irgendwo auf ihrer Festplatte finden. Das Programm wird dann in den Hauptspeicher kopiert werden und wird zu starten beginnen. Während das Programm läuft verlangt es nach Information, die in zusätzlichen Dateien auf der Festplatte gespeichert sind, welche das Betriebssystem finden und in den Hauptspeicher kopieren muss.

Wohin stellt ein Programm üblicherweise seine Ergebnisse um Sie langfristig zu speichern?

Dateien und das Betriebssystem

Üblicherweise in eine Datei auf dem Sekundärspeicher. Wenn die Datei noch nicht existiert, wird das Programm das Betriebssystem veranlassen sie zu erzeugen.

Gewöhnlich sind alle Sammlungen von Daten außerhalb des Hauptspeichers in Dateien organisiert. Es ist die Aufgabe des Betriebssystems alle diese Information auf die Reihe zu kriegen. Wenn das Computersystem Teil eines Netzwerks ist, kann die Aufgabe, den Überblick über alle Dateien auf allen Computern zu behalten, eine ganz schöne Aufgabe sein, und ist die gemeinsame Aufgabe aller beteiligten Betriebssysteme.

Applikationsprogramme (einschließlich der Programme, die Sie schreiben könnten) können nicht direkt Dateien lesen, schreiben, erzeugen oder löschen. Da das Betriebssystem alles verfolgen muss, fordern es alle anderen Programme auf die Handhabung von Dateiaufgaben zu übernehmen. Zum Beispiel angenommen ,dass ein Programm gerade eine Reihe von Zahlen berechnet hat und sie speichern muss. Es könnte das folgende tun:

  1. Das Programm: fordert das Betriebssystem auf eine Datei mit dem Namen ERGEBNIS.DAT zu erzeugen.
  2. Das Betriebssystem: bekommt die Anforderung; findet einen unbenutzten Abschnitt auf der Platte und erzeugt eine leere Datei. Dem Programm wird mitgeteilt, wenn das erledigt ist.
  3. Das Programm: fordert das Betriebssystem auf die Zahlen in der Datei zu speichern.
  4. Das Betriebssystem: bekommt die Zahlen aus dem Hauptspeicherbereich des Programms und schreibt sie in die Datei. Das Programm bekommt die Rückmeldung, wenn das geschehen ist.
  5. Das Programm: setzt seine Arbeit - mit was auch immer - fort.

Daher wenn ein Applikationsprogramm läuft, verlangt es ständig vom Betriebssystem die Durchführung von Dateiaufgaben (und andere Aufgaben) und wartet darauf, dass sie abgeschlossen werden. Wenn ein Programm vom Betriebssystem etwas zu tun verlangt, was das Dateisystem beschädigt, wird das Betriebssystem sich weigern es zu tun. Moderne Programme sind so geschrieben, dass sie Alternativen haben, wenn eine Anforderung verweigert wird. Ältere Programme wurden nicht in dieser Art geschrieben und laufen nicht allzu gut auf modernen Computern.

Dateitypen

Bei modernen Computersystemen kann nur das Betriebssystem direkt alles mit Dateien auf der Festplatte tun. Welche Auswirkungen hat dies auf:

  1. die Sicherheit des Systems?
    • Die Sicherheit hat zugenommen, da Programme, die versuchen gefährliche oder dumme Sachen mit Dateien zu tun, es nicht können. Sie müssen das Betriebssystem fragen, das nur sichere und vernünftige Dinge zulässt.
  2. Computerspiele?
    • Ältere Computerspiele haben die Handhabung von Dateien selbst durchgeführt, ohne das Betriebssystem zu fragen. (Das alte DOS war dumm und langsam und viele Applikationsprogramme ignorierten es.) Diese Spiele würden auf modernen Computern nicht mehr laufen.
  3. den Komfort bei der Erstellung von Programmen?
    • Die Programmerstellung ist leichter, da die Arbeit mit Dateien vom Betriebssystem erledigt wird.
 Dateitypen

Soweit es die Festplatte betrifft sind alle Dateien gleich. Auf der elektronischen Ebene gibt es keinen Unterschied zwischen einer Datei, die ein Programm enthält und einer Datei, die Daten enthält. Alle Dateien sind benannte Kollektionen von Bytes. Natürlich, für was die Dateien verwendet werden, das ist verschieden. Das Betriebssystem kann eine Programmdatei nehmen, sie in den Hauptspeicher kopieren und beginnen sie auszuführen. Das Betriebssystem kann eine Datendatei nehmen und deren Information einem laufenden Programm zur Verfügung stellen, wenn es danach verlangt.

Datei, ausführbare Datei, Namen

Häufig zeigt der letzte Teil eines Dateinamens (die Endung) für welchen Gebrauch die Datei verwendet wird. Zum Beispiel in "meineDaten.txt" bedeutet ".txt", dass es erwartet wird die Datei als Kollektion von Text zu verwenden, das heißt als Sammlung von Zeichen. Bei "Netscape.exe" bedeutet ".exe", dass die Datei "ausführbar" ist, das heißt ein Programm, dass bereit ist ausgeführt zu werden. Bei "Programm1.java" bedeutet ".java", dass die Datei ein Quellprogramm in der Sprache Java ist (über Quellprogramme wird es später in diesen Aufzeichnungen noch mehr geben). Für die Festplatte ist jede dieser Dateien das gleiche: eine Sammlung von Bytes.

Was passiert, wenn eine Datei für etwas anderes als ihren beabsichtigten Gebrauch verwendet wird? Was passiert zum Beispiel, wenn Sie versuchen eine ".EXE" Datei mit einem Textverarbeitungsprogramm zu öffnen?
