  • Quellcode-Datei.
    • Eine Textdatei in der Java Sprache, erzeugt durch einen Programmierer mit einem Texteditor.
      •
  • Bytecode-Datei.
    • Eine Datei bestehend aus Maschinensprache für die Java Virtuelle Maschine, erzeugt von einem Java Compiler.

Beispiel eines Quellprogramms

Hier ist das Quellprogramm (Quelldatei) aus dem vorherigen Kapitel. Der Zweck dieses Programms ist es die Zeichen Hello World! auf den Bildschirm auszugeben. 

class Hello
{
  public static void main ( String[] args )
  {
    System.out.println("Hello World!");
  }
}

Die Datei muss Hello.java genannt werden, um mit dem Klassennamen übereinzustimmen. Bei vielen Computersystemen ist die Groß-/Kleinschreibung des Dateinamens wichtig. (Daher, wenn die Datei hello.java heißt, mit einem kleinen h, wird es nicht funktionieren.) Bei allen Computern ist die Groß-/Kleinschreibung innerhalb des Programms wichtig. Die erste Zeile ... 

class Hello

... besagt, dass dieses Quellprogramm eine Klasse mit dem Namen "Hello" definieren wird. Eine Klasse ist ein Abschnitt eines Programms. Kleine Programme bestehen oft nur aus einer Klasse. (Eine bessere Definition von "Klasse" wird in späteren Kapiteln folgen.) Wenn das Programm kompiliert ist, wird der Compiler eine Datei aus Bytecode mit dem Namen Hello.class erzeugen.

Manche Klassen enthalten viele Zeilen. Alles, was eine Klasse ausmacht, wird zwischen die erste geschweifte Klammer ( { ) und die dazugehörige letzte geschweifte Klammer ( } ) gestellt.

Der Name der Klasse (und folglich der Name der Datei) ist Ihr Job. Der Name muss aus Buchstaben und Ziffern gebildet werden. Das erste Zeichen muss ein Buchstabe sein, und der Name darf keine Leerzeichen enthalten. Üblicherweise beginnen Klassennamen mit einem großen Buchstaben, aber das wird nicht vorausgesetzt. Die Quelldatei sollte immer mit ".java" in kleinen Buchstaben enden.
