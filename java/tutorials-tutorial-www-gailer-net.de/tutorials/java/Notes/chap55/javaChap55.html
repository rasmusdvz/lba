Swing und das AWT

In den USA wird "GUI" wie "gooey" ausgesprochen.

Application Windowing Toolkit AWT component

Eine grafische Benutzoberfläche besteht aus grafischen Komponenten, wie:

  • Fenster
  • Befehlsschaltflächen
  • Menüs
  • Textfeldern

Swing-Komponenten basieren auf den fundamentalen Klassen des Application Windowing Toolkit (AWT). Das AWT enthält auch einige ältere Komponenten, die nicht mehr verwendet werden. Swing-Komponenten sind Bestandteil von Java. Wenn Sie die neueste Java Version heruntrerladen, dann ist Swing ein Teil davon.

Die älteren AWT-Komponenten sind ähnlich wie die Swing-Komponenten, aber Swing-Komponenten sind vielseitiger. Auch ist Swing nahtlos in die Java Foundation Klassen integriert. Die Java Foundation Klassen stellen viele Klassen zur Verfügung, die für moderne Softwareentwicklung notwendig sind. Verwenden Sie Swing für alle neuen Programmcodes.

Ist eine Befehlsschaltfläche eine GUI Komponente?

Ereignisse

Ja.

Ein GUI Programm zeigt ein Interface, das grafische Komponenten enthält. Der Anwender kontrolliert die Applikation, indem er mit den grafischen Komponenten kommuniziert und dabei solche Dinge tut wie:

  • Auf eine Befehlsschaltfläche zu klicken, um eine Programmoption zu wählen.
  • Aus einem Menü auszuwählen.
  • Text in ein Textfeld einzugeben.
  • Eine Bildlaufleiste zu scrollen.

Eine Aktion, wie auf eine Befehlsschaltfläche zu klicken, wird ein Ereignis genannt. Während Sie diese Aufzeichnungen lesen erzeugen Sie Ereignisse, indem Sie die grafische Benutzeroberfläche Ihres Browsers verwenden.

Ereignisgesteuerte Programmierung

Klick!

event event-driven programming

Wenn Sie mit einer grafischen Komponente eine Aktion ausführen, erzeugen Sie ein Ereignis. In der ereignisgesteuerten Programmierung antwortet das Programm auf diese Ereignisse. Die Reihenfolge der Ereignisse wird vom Anwender bestimmt, nicht vom Programm.

Das unterscheidet sich von Programmen, bei denen die Interaktion mit dem Benutzer über die Konsole stattfindet. Bei diesen Programmen werden Eingabeaufforderungen auf die Konsole ausgegeben, und der Anwender antwortet auf diese Eingabeaufforderungen. Die Reihenfolge dieser Eingabeaufforderungen wird durch das Programm bestimmt.

Beim Programmieren mit Java bekommen Sie die GUI Komponenten, die Sie möchten, indem Sie sie anfordern. Das meiste der Arbeit wurde schon erledigt und ist im Paket Swing enthalten. Swing enthält Fenster, Fra windows, frames, Befehlsschaltfläches, menus and other Komponenten.

Ein Anwender kommuniziert mit einer GUI Applikation, indem er Ereignisse auslöst. Die Applikation muss auf die Ereignisse in der Reihenfolge in der sie eintreffen reagieren. Die Abblidung zeigt das. Jedes Mal wenn der Anwender mit einer Komponente etwas tut, wird ein Ereignis an die Applikation gesendet. Unterschiedliche Ereignisse werden an unterschiedliche Teile der Applikation gesendet. Die Applikation muss korrekt arbeiten, was auch immer der Anwender tut. (Üblicherweise ignoriert eine Applikation Ereignisse, die für sie nicht wichtig sind.)

Die drei Teile eines GUI Programms.

Bewegen Sie den Mauszeiger über das Abbildung. Ignoriert Ihr Browser dieses Ereignis?

Vielleicht. Manche Browser ignorieren dieses mouseOver Ergeignis. Die meisten Browser zeigen den Inhalt des title-Attributs an.

Bewegen Sie jetzt die Maus über die "next" Befehlsschaltfläche. Wird dieses Ereignis ignoriert?

Normalerweise nicht. Die meisten Browser ändern den Mauszeiger und zeigen als mouseOver Ereignis eine Hand an.

Natürlich, wenn Sie auf den Button klicken ist das ein anderes Ereignis und wird nicht ignoriert.

GUI program, three tasks

Ein GUI Programm besteht aus drei Arten von Software:

  1. Grafische Komponenten, die die grafische Benutzeroberfläche ausmachen.
  2. Listener-Methoden, die die Ereignisse empfangen und auf sie reagieren.
  3. Applikationsmethoden, die für den Anwender die nützliche Arbeit tun.

Die grafischen Komponenten sind Swing-Objekte. Gewöhnlich erweitern Sie sie, damit sie zu Ihrer Applikation passen.

Listener-Methoden sind Java Methoden, die Sie schreiben. Listener-Methoden antworten auf Ereignisse, indem sie Applikationsmethoden aufrufen.

Applikationsmethoden sind normale Java Methoden, die nützliche Dinge tun. Sie empfangen Daten von der grafische Benutzeroberfläche und senden Daten an die grafische Benutzeroberfläche, aber sind gewöhnlich nicht mit der grafische Benutzeroberfläche befasst.

Ihr Webbrowser hat Komponenten, wie den "zurück" und "vor" Button. Listener-Methoden empfangen diese Ereignisse und Applikationsmethoden tun die nützlichen Dinge, wie eine Seite zurück oder vorgehen.

Um eine GUI Applikation zu schreiben, halten Sie diese drei Teile eines Programms getrennt. Bei einem großen Softwareprojekt arbeiten komplette Teams von Programmierern an jedem dieser Teile.

Nicht-Software Geräte haben normalerweise ebenfalls eine Benutzerschnittstelle. Sie haben:

  1. Komponenten, die die Benutzerschnittstelle ausmachen.
    • Knöpfe, Befehlsschaltflächen, Schalter, Kipphebel u.a.
  2. Teile, die mit den Komponenten der Benutzerschnittstelle verbunden sind.
    • Diese "übersetzen" die Aktion des Anwenders in etwas, womit der Rest des Geräts etwas tun kann.
  3. Der Teil des Geräts, das die tatsächliche Arbeit tut.

Containerklassen

Ein Fenster.

container class

Ein GUI Programm besteht aus einer Sammlung grafischer Komponenten, die alle innerhalb eines oder mehrerer Fenster platziert sind. Die Komponenten sind in einem Fenster enthalten. Ein Container ist ein Objekt, das andere GUI Komponenten enthalten kann. Visuell gesehen ist ein Container ein Fläche auf dem Bildschirm und die Objekte, die er enthält, werden in ihm als kleinere Flächen angezeigt.

In der Java Terminologie ist ein Fenster (wie das Browserfenster, das Sie betrachten) ein Container. Buttons, Schieberegler, Icons und andere GUI Komponenten werden immer in einen Container gestellt.

Javaklassen (Rückblick)

Kann ein Container andere Container enthalten?

Ja. Sie sehen das, wenn das Hauptfenster eines Browsers Frames enthält, die wiederum andere Komponenten enthalten.

Die Component-Klassen, die das AWT und Swing ausmachen, sind miteinander über Vererbung verwandt. Die Klasse, die der ultimative Vorfahre aller anderen Klassen ist, wird Object genannt. Die Klasse Object definiert die grundsätzliche Natur aller Java Objekte. Vieles, was im Hintergrund passiert, wenn ein Java Programm läuft, wird durch Object definiert. AWT Hierarchie

Ja. Kindklassen der gleichen Elternklasse haben die Merkmale ihrer Elternklassen gemeinsam, aber jede Kindklasse hat zusätzliche Methoden und Variablen.

AWT Hierarchy

Das Java AWT (Abstract Windowing Toolkit) enthält die fundamentalen Klassen, die für die Konstruktion von GUIs verwendet werden. Die abstrakte Klasse Component ist die Basisklasse des AWTs. Viele AWT Klassen sind von ihr abgeleitet. Das sind die alten AWT Komponenten, die nicht mehr verwendet werden.

Einige der AWT Klassen, die von Component abgeleitet sind, sind Button, Canvas und Container.

Die Abbildung zeigt wie die Klassen des AWT und Swing zusammenpassen. Betrachten Sie die Abbildung, um einen Überblick zubekommen. Kehren Sie, während Sie diese Aufzeichnungen lesen, gelegentlich zurück.

Die JComponent-Klasse ist von Container abgeleitet und ist eine der Basisklassen von Swing. Die JFrame-Klasse ist von der Frame-Klasse abgeleitet. Sie ist gewöhnlicherweise der Hauptcontainer einer GUI Applikation.

Die JApplet-Klasse ist von der AWT Applet-Klasse abgeleitet und wird für moderne Applets verwendet.

(Rückblick:) Was ist eine Containerklasse?

JComponent-Klasse

Eine GUI Komponente, die von einer Containerklasse konstruiert wurde und die wieder andere GUI Komponenten enthalten kann.

JComponent class

Objekte der Container-Klasse oder Klassen, die auf Container basieren, haben gemeinsame Merkmale:

  • Sie können Komponenten enthalten.
  • Sie können selbst in Container gestellt werden.
  • Sie können auf dem Bildschirm angezeigt werden.
  • Sie erzeugen Ereignisse.
  • Sie registrieren Ereignisempfänger (Eventlisteners) usw.

Da alle Swingklassen Container als Vorfahren haben, haben alle Swingobjekte diese Merkmale.

Es gibt mehr Swingklassen als hier in der Abbildung gezeigt werden. Zum Beispiel gibt es mehrere Button Typen (wie JButton), die von AbstractButton abgeleitet sind.

Mit welchem Buchstaben beginnen viele Swingklassen?

Viele Swingklassen beginnen mit J (zum Beispiel JButton). Vergleichbare AWT Klassen beginnen nicht mit J (zum Beispiel Button).
