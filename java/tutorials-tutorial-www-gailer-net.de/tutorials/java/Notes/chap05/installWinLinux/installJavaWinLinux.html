Editor, IDE, Compiler

Mit jedem ASCII-Editor kann Java-Quellcode editiert werden, aber zumindest auf Syntax-Highlighting sollte nicht verzichtet werden.
Kompiliert wird zum Beispiel mit dem J2SE-Compiler von Sun.
Alternativ können komplexe IDEs (Integrated Developer Environment) eingesetzt werden, die einen Editor, Compiler und Debugger beinhalten und einige Arbeiten vereinfachen.
Um Java zu lernen, sollte mit einfachen Editoren (statt der IDEs) begonnen werden, um die Grundlagen zu verstehen.
  • IDEs:
  • Eclipse (Windows, Linux) (siehe auch Erste Schritte in Eclipse)
  • NetBeans (Windows, Linux, Solaris, AIX, HP/UX, Mac OS)
  • Java Studio Creator (Windows, Linux, Solaris, Mac OS)
  • Borland JBuilder (Windows, Linux, Solaris, Mac OS X).
  • IBM WebSphere Studio (Windows, Linux, Solaris, AIX, HP/UX)
  • Sun ONE Studio (früher Forte) (Windows, Linux, Solaris)
  • IntelliJ IDEA (Windows, Linux, Solaris, Mac OS X)
  • CodeGuide (Windows, Linux, Solaris, Mac OS X)
  • Xinox JCreator (Windows)
  • Editoren:
  • Java: Z.B. jEdit.
  • Windows: Z.B. UltraEdit, TextPad, EditPlus und EditPad.
  • Unix: Z.B. Emacs, XEmacs, GNU-Emacs und NEdit.
  • Mac: Z.B. BBEdit.
  • Native Compiler (versuchen Executables (z.B. .exe für Windows) zu erzeugen, was aber problematisch ist):
  • Excelsior JET
  • http://www.geocities.com/marcoschmidt.geo/jcomp.html#native
  • http://www.bearcave.com/software/java/comp_java.html


Windows:
Installation des J2SE SDK (auch J2SDK, Java 2 SDK oder JDK genannt)

  1. Falls Java-Applets für Web-Browser erstellt werden sollen: Beachten Sie die Bemerkungen zu Java-Versionen in java-plugin.htm.

  2. Falls Sie bereits Java installiert haben, überprüfen Sie, ob es vielleicht aktuel genug ist. Lassen Sie sich die Version anzeigen mit:

    java -version

    Überprüfen Sie, ob Sie nur das JRE oder das komplette SDK (JDK) installiert haben (außer 'java.exe' muss es auch 'javac.exe' geben).

  3. Bevor Sie Java installieren, sollten Sie bereits installierte Versionen vorher deinstallieren. Unter Windows XP erfolgt das folgendermaßen: 'Start' | 'Systemsteuerung' | 'Software' | 'Java ... Entfernen'.

  4. Downloaden Sie von http://java.sun.com/j2se entweder

    für J2SE SDK 1.4z.B. die Dateien 'j2sdk-1_4_2_08-windows-i586-p.exe' und 'j2sdk-1_4_2-doc.zip'
    oder
    für J2SE JDK 5.xz.B. die Dateien 'jdk-1_5_0_03-windows-i586-p.exe' und 'jdk-1_5_0-doc.zip'.

  5. Starten Sie die ausführbare Windows-Installationsdatei. Wählen Sie ein passendes Zielverzeichnis bei "Install to" bzw. "Destination Folder". Im Folgenden wird davon ausgegangen, dass nicht die Vorgabe, sondern 'C:\Java\j2sdk' eingestellt wird (unter das Verzeichnis 'C:\Java' können so später weitere Java-Module installiert werden und das Weglassen der Versionsnummer hinter 'C:\Java\j2sdk' erleichtert den späteren Wechsel auf eine andere Version).
  6. Expandierung der Doku-Zip-Datei nach 'C:\Java\j2sdk'.
  7. Falls Sie Ant verwenden wollen: Downloaden Sie die Ant-Installationsdatei (z.B. 'apache-ant-1.6.5-bin.zip') von http://ant.apache.org. Entpacken Sie das Ant-Archiv in ein temporäres Verzeichnis, kopieren Sie den darin enthaltenen Ant-Ordner (z.B. 'apache-ant-1.6.5') zum Beispiel nach 'C:\Java' und umbenennen Sie ihn in 'C:\Java\ant'.

  8. Jetzt müssen folgende Kommandos ein erfolgreiches Ergebnis liefern:

    C:\Java\j2sdk\bin\java.exe -version

    C:\Java\j2sdk\bin\javac.exe -help

    start C:\Java\j2sdk\docs\index.html

    9. C:\Java\ant\bin\ant.bat -h

  9. Einige Anwendungen erwarten korrekt gesetzte Umgebungsvariablen (Environment-Variablen) (passen Sie die Pfadangaben an Ihre Java-SDK- und Ant-Verzeichnisse an):

    Benutzervariablen setzen:
    JAVA_HOMEC:\Java\j2sdk
    Systemvariablen erweitern um:
    Path<vorherigerPath>;C:\Java\j2sdk\bin;C:\Java\ant\bin

    Folgendermaßen ändern Sie die Umgebungsvariablen:

    Windows XP'Start' | rechter Mausklick auf 'Arbeitsplatz' | 'Eigenschaften' | 'Erweitert' | 'Umgebungsvariablen'
    Windows 2000'Start' | 'Einstellungen' | 'Systemsteuerung' | 'System' | 'Erweitert' | 'Umgebungsvariablen...'
    Windows 98/MeC:\Autoexec.bat:
    set JAVA_HOME=C:\Java\j2sdk
    set CLASSPATH=
    set path=%JAVA_HOME%\bin;%path%
    Linuxsiehe linux.htm#Umgebungsvariablen

  10. Jetzt müssen folgende Kommandos ein erfolgreiches Ergebnis liefern. Öffnen Sie ein neues Kommandozeilenfenster (damit die Änderung der Umgebungsvariablen wirksam wird) und geben Sie ein:

    set JAVA_HOME

    java -version

    javac -help

    ant -h

  11. Für ältere Windows-Versionen empfiehlt es sich für einige Batch-Dateien (z.B. mit langen CLASSPATH-Anweisungen) das Konsolen-Environment zu vergrößern, beispielsweise so:

    Windows XP
    Windows 2000     		Keine Änderung nötig
    Windows MeIn den "Eigenschaften von MS-DOS-Eingabeaufforderung" unter "Arbeitsspeicher" den "Umgebungsspeicher" auf 3072 setzen
    Windows 98C:\Config.sys:
    shell=C:\Command.com C:\ /P/E:3000



SuSE Linux 7.x und 8.x:
Installation des J2SE 1.4 (auch J2SDK 1.4 oder Java 2 SDK 1.4 genannt)

  1. Falls Java-Applets für Web-Browser erstellt werden sollen: Bemerkungen zu Java-Versionen in java-plugin.htm beachten.
  2. Java kann in vielen Linux-Distributionen aus der Distribution installiert werden. Allerdings ist man dann auf die vorhandene Java-Version festgelegt. Deshalb und für spätere Updates ist hier die Installation per Internet-Download beschrieben.
  3. SuSE Linux 7.x/8.x Personal reicht aus, es wird nicht die Professional-Version benötigt. Im Folgenden wird davon ausgegangen, dass Linux ähnlich installiert ist, wie in linux.htm beschrieben.
  4. Falls eventuell bereits installierte ältere Java-Versionen noch benötigt werden, können sie installiert bleiben. Falls nicht, sollten sie deinstalliert werden. Unter SuSE 8.x folgendermaßen:
    'YaST2' | 'Software' | 'Software löschen',
    jeweils die beiden folgenden 'Gruppen'/'Pakete' 'Abwählen' (bis 'd' oder nichts davor steht):
      'Entwicklung/Programmiersprache/Java' / 'java'
      'Entwicklung/Programmiersprache/Java' / 'java2'
  5. Download von J2SE 1.4 (j2sdk-1_4_0-linux-rpm.bin und j2sdk-1_4_0-doc.zip) von
    http://java.sun.com/j2se/1.4. Dateien in ein Verzeichnis mit Schreibrechten kopieren (z.B. /tmp)
    (Dateinamen weichen je nach Version und Bugfix-Level etwas ab).
  6. Überprüfen, ob es sich um ein Shell-Skript handelt, mit:
    file j2sdk-1_4_0-linux-rpm.bin
  7. Das Skript ausführen und in Lizenzbedingungen mit Leertaste weiterblättern bis mit "yes" bestätigt werden kann:
    sh j2sdk-1_4_0-linux-rpm.bin
  8. Nachsehen, wie entpackte Datei heißt (z.B. "j2sdk-1_4_0-linux.rpm"):
    ls -ltr
  9. Diese .rpm-Datei installieren mit:
    rpm -i j2sdk-1_4_0-linux.rpm
  10. Expandierung der Doku-Zip-Datei nach "/usr/java/j2sdk1.4.0".
  11. Ansehen: /usr/java/j2sdk1.4.0/docs/index.html
  12. Nachsehen, in welches Verzeichnis das J2SE installiert wurde (z.B. "/usr/java/j2sdk1.4.0") (notfalls im Dateisystem nach "javac" suchen, aber dabei nicht von eventuell anderen installierten Java-Versionen täuschen lassen).
    Das bin-Unterverzeichnis dieses Java-Root-Verzeichnisses muss im 'PATH' und das Java-Root-Verzeichnis auch in anderen Umgebungsvariablen eingetragen sein.
    Das weitere Vorgehen hängt davon ab, ob noch andere Java-Versionen installiert bleiben sollen. Hier wird angenommen, dass zusätzlich zum neuen J2SDK noch das ältere JDK 1.1.8 installiert sei, aber alle Umgebungsvariablen auf das neue J2SDK umgestellt werden sollen.
    Die aktuellen Java-Umgebungsvariablen können dann so ermittelt und angezeigt werden:
    set | grep -i java
      JAVA_BINDIR=/usr/lib/java/bin
      JAVA_HOME=/usr/lib/java
      JDK_HOME=/usr/lib/java
      JRE_HOME=/usr/lib/java
      PATH=...:/usr/lib/java/bin:...
     Ein temporäres Umstellen der Umgebungsvariablen nur für das aktuelle Kommandozeilenfenster und dessen Tochterprozesse ist recht einfach. Für die 'PATH'-Variable zum Beispiel so:
    PATH=/usr/java/j2sdk1.4.0/bin:$PATH
    export PATH
    echo $PATH
     Eine dauerhafte Umstellung der Umgebungsvariablen ist etwas komplizierter und muss je nach Linux-Distribution unterschiedlich erfolgen. Unter SuSE Linux 7.x/8.x wird im Verzeichnis '/etc' eine neue Datei mit dem Namen 'profile.local' und folgendem Inhalt erzeugt:
    JAVA_BINDIR=/usr/java/j2sdk1.4.0/bin
    JAVA_HOME=/usr/java/j2sdk1.4.0
    JDK_HOME=/usr/java/j2sdk1.4.0
    JRE_HOME=/usr/java/j2sdk1.4.0
    PATH=`echo $PATH|sed 's|:/usr/lib/java/bin|:/usr/java/j2sdk1.4.0/bin|'`
    export JAVA_BINDIR
    export JAVA_HOME
    export JDK_HOME
    export JRE_HOME
    export PATH
