 Funktion des Schlosses

    • Drei.
  • Ist die Reihenfolge der Auswahl Ihrer Zahlen wichtig?
    • Ja.
  • Wie ist Ihre Zahlenkombination?

  • Gibt es etwas in dem Spind, das ich brauchen könnte?

Ein typisches Zahlenschloss wird sich erst dann öffnen, wenn die Wählscheibe in der richtigen Reihenfolge auf alle 3 Kombinationen gedreht wurde. Zum Beispiel würde Die Kombination "6-12-30" so verwendet werden:

  1. Drehen Sie die Wählscheibe zweimal im Uhrzeigersinn ganz herum.
  2. Drehen Sie die Wählscheibe im Uhrzeigersinn auf "6"
  3. Drehen Sie die Wählscheibe im Uhrzeigersinn auf "12"
  4. Drehen Sie die Wählscheibe im Uhrzeigersinn auf "30"
  5. Ziehen Sie das Schloss auf.

In unserem Programm vernachlässigen wir die Drehrichtung. Das Schloss wird sich öffnen lassen, wenn der Anwender die richtigen drei Zahlen in der richtigen Reihenfolge eingegeben hat.

Bekommen Sie unmittelbar nach der Eingabe einen Hinweis auf eine falsche Zahl?

Ablaufplan des Zahlenschloss-Programmes

Nein. Das würde es Unbefugten zu leicht machen, die Kombination herauszufinden.

program, combination lock

Erst wenn Sie alle drei Zahlen eingegeben haben, erfahren Sie, ob sie korrekt waren. Wenn das Schloss sich nicht öffnet, müssen Sie es noch einmal ganz von vorne probieren. Der Ablaufplan zeigt, wie das Schloss funktioniert.

Der Ablaufplan verwendet eine Variable korrekt, die zu Beginn mit true initialisiert wird, aber auf false gesetzt wird, sobald eine falsche Zahl eingegeben wird. Diese Variable wird nach jeder der drei Eingaben geprüft, um herauszufinden, ob das Schloss aufgehen soll oder nicht.

Überprüfen Sie den Ablaufplan. Kann der Benutzer herausfinden, welche seiner Zahlen falsch war, wenn sich das Schloss nicht öffnet?

Grundgerüst des Programmes

Nein. Alles was der Anwender weiss, ist das etwas falsch ist.

Betrachten Sie noch einmal den Ablaufplan und sehen Sie sich an, wie die boolsche Variable korrekt arbeitet. Die if-Anweisung im Programm ist korrekt geschrieben. Die boolsche Variable korrekt wird entweder true oder false sein. Das ist alles, was eine if-Anweisung braucht. Sie brauchen folgendes nicht zu tun: 

if ( korrekt == true )

denn die Ausführung der booleschen Anweisung wird lediglich den Wert ergeben, den die Variable korrekt bereits hat.



Wird korrekt jemals auf true zurückgesetzt werden, wenn es einmal auf false gesetzt wurde?

Vollständiges Programm

Das vollständige Programm verwendet für die nächsten zwei Kombinationen fast genau den gleichen Code wie für die erste Kombination.

Hier ist das vollständige Programm. Die Unterschiede im Code sind für jede der drei Kombinationen farbig markiert. Das Erkennen von Gemeinsamkeiten wie diesen zwischen Programmteilen führt oft zu besseren Programmen. 

import java.io.*;

class Zahlenschloss
{
  public static void main ( String[] args ) throws IOException
  {
    int ersteZahl  =  6,
        zweiteZahl = 12,
        dritteZahl = 30;  // Die Kombination
    int zahl;             // Benutzer-Eingabe

    BufferedReader stdin = new BufferedReader(
        new InputStreamReader( System.in ) );
    String input;

    boolean korrekt = true;

    //Erste Zahl
    System.out.println("Geben Sie die erste Zahl ein: ");
    input = stdin.readLine();
    zahl  = Integer.parseInt( input );

    if ( zahl != ersteZahl )
      korrekt = false ;

    //Zweite Zahl
    System.out.println("Geben Sie die zweite Zahl ein: ");
    input = stdin.readLine();
    zahl  = Integer.parseInt( input );

    if ( zahl != zweiteZahl )
      korrekt = false  ;

    //Dritte Zahl
    System.out.println("Geben Sie die dritte Zahl ein: ");
    input = stdin.readLine();
    zahl  = Integer.parseInt( input );

    if ( zahl != dritteZahl )
      korrekt = false  ;

    //Ergebnis
    if ( korrekt )
      System.out.println("Das Schloss öffnet sich.");
    else
      System.out.println("Das Schloss bleibt zu.");
  }
}

Nein.

Sehen Sie sich den ersten Kasten im Schleifen-Körper an (der, in dem steht "user tries a combination..."). Ist das Programm dafür bereits geschrieben?

Hauptschleife

Ja. Die main()-Methode der vorherigen Version des Programmes tut genau das (nach ein paar kleinen Änderungen).

Hier sehen Sie den Hauptteil des neuen Programmes. Prüfen Sie, ob es mit dem Ablaufplan übereinstimmt. Denken Sie daran, dass !offen bedeutet NOT offen, dass !true gleich false und !false gleich true ist. 

class ZahlenschlossSchleife
{
  public static void main( String[] args )
  {
    int     versuch  = 0;
    boolean offen    = false;

    while ( versuch < 3 && !offen )
    {
      // Testen einer Kombination, wenn richtig: Variable offen auf "true" setzen

      versuch = versuch + 1;
    }

  }
}

Tut 

while ( versuch < 3 && !offen )

das was wir wollen?

Boolscher Ausdruck

Tut 

while ( versuch < 3 && !offen )

das was wir wollen?

Ja, denn der boolesche Ausdruck bedeutet: 

while ( der Benutzer hat weniger als 3 Versuche gemacht UND das Schloss ist NICHT offen )

Der Boolsche Ausdruck 

 versuch < 3 && !offen

wird true ergeben, wenn der Benutzer die erste oder eine weitere Möglichkeit bekommt, und er wird false ergeben, wenn der Benutzer keine weitere Möglichkeit mehr bekommt. Die Tabelle zeigt die Funktionsweise:

Anzahl der Versuche offen versuch < 3 !offen versuch < 3 && !offen Weiterer Versuch?
0 false true true true && true true
1 false true true true && true true
1 true true false true && false false
2 false true true true && true true
2 true true false true && false false
3 false ? ? ? && ? ????
3 true ? ? ? && ? ????

Die Logik in einem Computer Programm ist häufig so kompliziert (oder noch komplizierter!) wie diese. Es könnte notwendig sein, solch eine Übersicht zu erstellen, um zu prüfen, ob die Logik korrekt ist.

(Denkaufgabe: ) Könnte dieses Programm auch kürzer sein?

Beispiel

Ja. Das Programm könnte ohne if-Anweisungen geschrieben werden. Es könnte nur eine einzige while-Anweisung benutzen, die einen ziemlich komplizierten booleschen Ausdruck überprüft. Der Benutzer würde alle drei Zahlen eingeben, bevor sie überprüft werden.

Es würde eine nützliche Übung für Sie sein, wenn Sie das Programm wie vorgeschlagen verkürzen würden. Programmierer arbeiten üblicherweiser an einem Programm weiter, wenn es zum ersten Mal die Anforderungen erfüllt hat. Oft schrumpft ein Programm in seiner Grösse und arbeitet trotzdem wie gewünscht weiter! (Aber ein Programm sollte nicht auf Kosten von Einfachheit und Klarheit verkürzt werden.)
